\chapter{Conclusion}
\section{Summary}
This thesis investigated how an AI agent can be integrated as a first-class collaborator in a web-based optimization platform for Mixed-Integer Linear Programming (MILP). The core objective was not to add a detached assistant, but to embed an execution-capable agent directly into the existing collaborative modeling workflow.

The implemented architecture combines: (i) in-modeler chat interaction, (ii) stateful LangGraph-based orchestration, (iii) a browser automation bridge using Puppeteer for environment-side execution, and (iv) human-in-the-loop supervision with iterative feedback. This design allows the agent to operate on the current shared workspace state, generate context-aware model updates, and apply validated changes through the same collaboration channel used by human participants.

With respect to the research questions introduced in this thesis, the work provides the following contributions:
\begin{itemize}
    \item \textbf{Reduction of modeling effort:} The architecture operationalizes natural-language-driven model editing and establishes an evaluation framework for measuring manual versus AI-assisted modeling speed.
    \item \textbf{Lowering entry barriers:} The integrated assistant provides on-demand support inside the existing modeling interface, reducing dependence on deep prior platform expertise.
    \item \textbf{Improved collaborative quality under supervision:} The system formalizes a stateful proposal-review loop where users retain final authority over acceptance, rejection, and refinement of generated updates.
\end{itemize}

Overall, the thesis demonstrates that AI support in optimization modeling is most effective when it is stateful, environment-aware, and explicitly supervised, rather than purely autonomous or prompt-only.

\section{Limitations and Future Work}
Although the prototype demonstrates feasibility and end-to-end integration, several limitations remain and motivate future work.

\subsection{Current Limitations}
\begin{itemize}
    \item The implementation currently prioritizes research prototyping and architectural integration over production hardening.
    \item Evaluation results depend on the selected task set and participant distribution; broader empirical studies are required for stronger generalization claims.
    \item Reliability can be further improved by expanding and standardizing validator coverage across all model transformation cases.
\end{itemize}

\subsection{Future Directions}
\begin{itemize}
    \item \textbf{Extended validation pipeline:} Integrate modeler-native validators as a stricter pre-commit gate across all update paths, with richer error diagnostics for iterative correction.
    \item \textbf{Larger-scale evaluation:} Conduct broader user studies with diverse optimization domains and statistically grounded comparisons of efficiency, accessibility, and quality metrics.
    \item \textbf{Operational hardening:} Improve observability, service robustness, and deployment standardization for long-running collaborative sessions.
    \item \textbf{Interaction refinement:} Add richer UI feedback for agent state transitions (generation, validation, waiting-for-approval) to further improve transparency and trust.
\end{itemize}

In conclusion, this work establishes a practical and extensible foundation for human-AI co-modeling in optimization platforms and shows that first-class collaborative agents are a viable direction for improving both productivity and accessibility in technical modeling workflows.
